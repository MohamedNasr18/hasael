Task 3 — Farm owner: edit and delete farm
On the farm owner's profile page or my-farms page, each farm card must have two action buttons: تعديل and حذف.
Edit:

Clicking "تعديل" opens a modal or navigates to an edit form pre-filled with the farm's existing data.
On submit → PATCH /api/farms/:id with only the changed fields.
On success: refresh the farms list and show a success message.

Delete:

Clicking "حذف" shows a confirmation dialog: "هل أنت متأكد من حذف هذه المزرعة؟" with confirm and cancel buttons.
On confirm → DELETE /api/farms/:id.
On success: remove the farm card from the UI immediately without full page reload.

Both buttons are only visible when user._id === farm.owner._id.

Task 4 — Investor: invest button on farm card
When the logged-in user's active_role is investor, each farm card in the feed must show an "استثمر الآن" button.
Behavior:

Clicking the button → POST /api/investments/:farmId with body: { type: "express_interest" }.
No payment form, no validation modal — just send the request directly.
On success: change the button to "تم الإرسال ✓" and disable it so it cannot be clicked again.
On error: show a small error message under the button (e.g. "حدث خطأ، حاول مرة أخرى").
If the user has already submitted an investment for this farm (check GET /api/investments/my on page load), show the button as disabled with text "تم الإرسال ✓" from the start.
